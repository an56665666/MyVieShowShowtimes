# 開發規範與工作流程

## 語言與文件規範（Language & Documentation）

### 人類溝通與文件（Human-readable content）

本專案所有人類閱讀內容一律使用繁體中文，包含但不限於：

- Spec 文件
- 設計文件（Design Docs）
- 程式碼註解（Comments）
- Commit message（如有規範）
- API 文件說明
- README / 開發文件
- AI 與開發者之間的對話內容

### AI Agent 規範

AI Agent（包含 ChatGPT、GitHub Copilot 等）必須：

- 使用繁體中文回應與產出內容
- 不可使用簡體中文
- 不可中英混雜（除非為專有名詞、API 名稱或程式碼）
